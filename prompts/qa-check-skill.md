# QA Check Skill

## 适用场景
用于在商品发布前检查标题、五点卖点、FAQ、详情页结构和最终输出，发现影响阿里国际站搜索、合规和询盘转化的问题。

## 输入字段
- 产品名称：{{productName}}
- 产品中心词：{{productCore}}
- 核心关键词：{{coreKeyword}}
- 品牌/营销词：{{brandMarketing}}
- 属性词：{{attributeWords}}
- 关键词变体：{{keywordVariants}}
- 应用场景：{{scenarios}}
- 材质：{{material}}
- 规格：{{specs}}
- 功能：{{functions}}
- 定制能力：{{customization}}
- MOQ：{{moq}}
- 样品：{{sample}}
- 交期：{{leadTime}}
- 包装：{{packaging}}
- 认证：{{certifications}}
- 目标国家：{{targetCountries}}
- 目标买家：{{targetBuyers}}
- 补充说明：{{notes}}

## 执行步骤
1. 检查堆词：标题和卖点是否重复堆叠同义关键词，是否影响可读性。
2. 检查虚假承诺：是否出现未提供依据的认证、质保、交期、价格、售后或绝对化表达。
3. 检查信息缺失：MOQ、样品、定制、交期、包装、认证、付款、售后是否缺少必要说明。
4. 检查卖点空泛：是否只有形容词，没有采购价值、参数依据或应用场景。
5. 检查 FAQ 编造：FAQ 是否编造付款条款、售后政策、认证编号或未给出的具体数据。
6. 按严重程度给出通过、警告、需修改三类结果。
7. 给出可执行修改建议。

## 输出格式
```text
质检总评：

通过项：
- ...

警告项：
- 问题：
  建议：

需修改项：
- 问题：
  建议：

缺失字段：
- ...
```

## 禁止事项
- 禁止只输出“通过”但不给检查依据。
- 禁止忽略 FAQ 编造、虚假承诺和关键词堆砌。
- 禁止将缺失信息自动补全为具体数据。
- 禁止给出无法执行的泛泛建议。
