# Detail Page Skill

## 适用场景
用于规划阿里国际站商品详情页结构，帮助运营把生成内容交给美工、详情页编辑或 AI 设计工具继续制作页面。

## 输入字段
- 产品名称：{{productName}}
- 产品中心词：{{productCore}}
- 核心关键词：{{coreKeyword}}
- 品牌/营销词：{{brandMarketing}}
- 属性词：{{attributeWords}}
- 应用场景：{{scenarios}}
- 材质：{{material}}
- 规格：{{specs}}
- 功能：{{functions}}
- 定制能力：{{customization}}
- MOQ：{{moq}}
- 样品：{{sample}}
- 交期：{{leadTime}}
- 包装：{{packaging}}
- 认证：{{certifications}}
- 目标国家：{{targetCountries}}
- 目标买家：{{targetBuyers}}
- 补充说明：{{notes}}

## 执行步骤
1. 设计首屏模块，突出产品定位、核心关键词和询盘行动。
2. 设计核心卖点模块，说明材质、功能、规格和使用价值。
3. 设计应用场景模块，连接目标国家、行业用途和买家角色。
4. 设计定制能力模块，展示 logo、颜色、规格、包装、OEM/ODM 等能力。
5. 设计采购信息模块，展示 MOQ、样品、交期、包装和发货准备信息。
6. 设计信任背书模块，展示认证、质检、工厂能力或服务流程。
7. 设计 FAQ 或询盘引导模块，降低买家发起询盘前的阻力。

## 输出格式
```text
1. 模块名称
- 目标：
- 内容：
- 图片/素材建议：
- 询盘引导：
```

## 禁止事项
- 禁止只输出泛泛的页面栏目名。
- 禁止虚构工厂规模、客户案例、认证、检测报告或合作品牌。
- 禁止把详情页写成零售落地页，必须服务 B2B 询盘。
- 禁止遗漏定制、采购信息和信任背书模块。
