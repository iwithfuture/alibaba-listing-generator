# AGENTS.md

## 项目目标
把本项目维护成一个稳定的“阿里国际站商品发布工作流”工具：用户输入产品资料后，系统按固定流程生成资料结构化、标题、五点卖点、FAQ、详情页结构、质检结果和最终发布版。

## 目录结构
- `index.html`：单页工具界面，包含输入表单、案例库控件和输出模块。
- `styles.css`：页面视觉样式和响应式布局。
- `app.js`：前端数据结构、工作流执行、质检规则、案例加载、复制功能。
- `prompts/`：可维护的模块化提示词，不要把提示词正文写进页面组件。
- `examples/`：案例库 JSON，用于一键加载和回归测试工作流稳定性。
- `README.md`：用户侧说明。
- `vercel.json`：Vercel 静态部署配置。

## Prompt 文件用途
- `prompts/product-brief.md`：把用户输入整理为标准 ProductBrief。
- `prompts/title-skill.md`：生成阿里国际站商品标题。
- `prompts/bullet-skill.md`：生成 B2B 采购价值导向的五点卖点。
- `prompts/faq-skill.md`：生成采购 FAQ。
- `prompts/detail-page-skill.md`：生成详情页结构。
- `prompts/qa-check-skill.md`：检查生成结果的合规性、完整性和 B2B 采购逻辑。
- `prompts/final-output-skill.md`：整理最终可复制的发布版内容。

## 工作流执行顺序
1. 读取表单字段，创建 `ProductBrief`。
2. 资料结构化，输出产品定位、关键词、应用场景、采购信息和缺失字段。
3. 生成标题。
4. 生成五点卖点。
5. 生成 FAQ。
6. 生成详情页结构。
7. 执行质检。
8. 生成最终发布版。

## 标题公式
标题必须遵守：

`品牌/营销词 + 属性词 + 产品中心词 + 核心关键词变体 + 应用场景`

要求：
- 产品中心词只保留一个主线。
- 属性词选择 2-4 个最能影响采购判断的词。
- 核心关键词变体不能和产品中心词、核心关键词机械重复。
- 未提供品牌时，可用 `Custom`、`Wholesale`、`OEM`、`Factory Supply`，不得虚构品牌名。

## 五点卖点规则
- 必须按 B2B 采购价值写，不写亚马逊式零售情绪文案。
- 五点建议顺序：采购适配、产品价值、定制能力、订单执行、市场信任。
- 每一点必须说明买家收益，不能只罗列参数。
- 必须覆盖可验证信息：材质、规格、功能、MOQ、样品、交期、包装、认证、定制能力中可用的字段。
- 缺失信息只能写“可确认 / 待确认 / 可沟通”，不得编造。

## FAQ 规则
FAQ 必须围绕：
- MOQ
- 样品
- 定制
- 交期
- 包装
- 认证
- 付款
- 售后

规则：
- 未提供付款条款时，只能写“订单数量、定制范围和交付条款确认后沟通”。
- 未提供售后政策时，只能写“按最终订单协议和检验标准确认支持范围”。
- 未提供认证时，不得写具体认证名称、编号或测试报告。

## 质检规则
质检必须检查：
- 标题是否堆词。
- 标题是否只有一个产品中心词。
- 是否编造认证、销量、排名、客户案例。
- 是否使用 `Best`、`No.1`、`Guaranteed`、`100% Perfect` 等风险词。
- 五点卖点是否空泛，是否缺少参数、场景或采购价值支撑。
- FAQ 是否出现输入资料未提供的信息。
- 是否缺少 MOQ、样品、交期、包装、定制能力等关键 B2B 信息。
- 是否符合阿里国际站 B2B 买家采购逻辑。

质检输出必须包含：
- `质检总评`
- `通过项`
- `警告项`
- `需修改项`
- `资料完整度`
- `待补充字段`

## 禁止事项
- 禁止把 prompt 正文重新写死进 `index.html` 或 UI 组件。
- 禁止编造认证、销量、排名、客户案例、合作品牌、质保年限、付款条款。
- 禁止使用无法证明的绝对化表达。
- 禁止为了通过质检而删除风险提示。
- 禁止破坏案例库加载、重新生成和复制功能。
- 禁止提交 `.vercel/` 本地配置目录。

## 必须运行的检查命令
完成任何代码或提示词改动后，至少运行：

```bash
node --check app.js
```

完成案例库改动后，还要运行：

```powershell
Get-ChildItem examples\*.json | ForEach-Object { Get-Content -Raw -Encoding UTF8 $_.FullName | ConvertFrom-Json | Out-Null }; 'examples ok'
```

涉及页面交互、案例库、复制或工作流逻辑时，还必须启动本地预览并在浏览器中验证：

```bash
npx serve .
```

如果不使用 `npx serve`，可用任意静态服务器，但必须确认页面能通过 HTTP 访问，且 `prompts/` 和 `examples/` 能被正常加载。
